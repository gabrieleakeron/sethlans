Code quality
CodeScene, Codacy, nessuno
Non configurato

Provider

Il nome esatto del progetto come appare in CodeScene.

Usato nel bind-mount del container Docker CodeScene. Lascia vuoto per usare la directory corrente.

Crea un token su codescene.io → Account → API Access. Passato all'MCP via CS_ACCESS_TOKEN.

Comando MCP proposto (CodeScene) 
claude mcp add codescene -s local \
  -e CS_ACCESS_TOKEN='${CS_ACCESS_TOKEN}' \
  -e CS_ONPREM_URL='${CS_ONPREM_URL}' \
  -- docker run -i --rm \
  -e CS_ACCESS_TOKEN -e CS_ONPREM_URL \
  -e CS_MOUNT_PATH= \
  --mount type=bind,src=,dst=/mount/,ro \
  codescene/codescene-mcp
Nessun tool di code quality configurato: nessun token richiesto, nessun MCP registrato. In project-profile.yaml viene registrato codeQuality.provider: none.
